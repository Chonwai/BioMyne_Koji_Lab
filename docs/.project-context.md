# Project Context Snapshot
Last Updated: 2026-07-07
Scope: full

---

## Tech Stack
- **Runtime:** Python 3 (scripts), Bash (pipeline orchestration), Docker Compose
- **Agent Framework:** Hermes Agent v0.18.0 (NousResearch, MIT, Python) — Docker image `nousresearch/hermes-agent:latest`
- **Local LLM:** Ollama — primary model Qwen 3.6 (27B Q4 / 35B MLX)
- **Web Scraping:** Firecrawl (API key, 1K free credits/month)
- **Database:** Supabase Cloud — PostgreSQL 15+ with pgvector extension, pgcrypto
- **Observability:** Langfuse v2 (Docker, port 3001)
- **Delivery:** Telegram (primary), Slack/Notion (configured but not yet validated)
- **No ORM** — raw SQL via Supabase REST API + `curl` helper
- **No web frontend** in Phase 1 (`biomyne-koji-api/` and `biomyne-koji-dashboard/` are empty scaffolds)

## DB Entities (Tables)
| Table | Description | Schema File |
|---|---|---|
| `sources` | Biotech news/journal sources to crawl | `sql/001_phase1_core_schema.sql` |
| `crawl_runs` | Pipeline execution history (manual/scheduled) | same |
| `articles` | Scraped + analyzed articles with priority & confidence | same |
| `entities` | Named entities extracted from articles (unique by name+type) | same |
| `article_entities` | M:N join — article-to-entity with role & confidence | same |
| `delivery_logs` | Digest delivery audit trail per channel | same |

## Naming Conventions
- **DB columns:** `snake_case` — evidence: `source_id`, `crawl_run_id`, `priority_level`, `confidence_score`, `created_at`, `article_count`, `delivered_at`
- **DB tables:** `snake_case` — evidence: `crawl_runs`, `article_entities`, `delivery_logs`
- **Files:** `kebab-case` — evidence: `run_pipeline.sh`, `smoke_test.sh`, `validate_sources.py`, `bootstrap_ollama.sh`
- **SQL migration files:** `NNN_description.sql` zero-padded numeric prefix
- **Config keys / env vars:** `UPPER_SNAKE_CASE` — evidence: `HERMES_URL`, `SUPABASE_SERVICE_ROLE_KEY`, `OLLAMA_MODEL_TAG`

## Pipeline Flow
```
Supabase sources → [source-intake] → Firecrawl scrape → [article-analysis] → Supabase articles/entities → [digest-publish] → Telegram/Slack
```
Manual execution: `bash ops/scripts/run_pipeline.sh` (requires Hermes Gateway running via Docker).

## Known Issues & Constraints
- **bash 3.2 bug:** macOS built-in bash 3.2 has a parsing bug with `$(...)` + `"..."` + `<<<` combinations (see `docs/hotfixes/2026-07-07-pipeline-python-syntaxerror.md`). Multi-line Python code must live in standalone `.py` scripts, not inline `-c` strings.
- **No scheduled runs yet:** `biotech-worker` profile has `schedule.enabled: false` — runs are manual-only until smoke test gate passes.
- **Phase 1 scope gate:** If work requires GraphDB/Neo4j, FastAPI BFF, Next.js dashboard, or multi-user auth → escalate, do not widen scope.
